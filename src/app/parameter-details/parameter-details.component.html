Parameter Details

The route for this page has no configuration to handle incoming parameters. The route configuration in app.module.ts is simply this:
path:'parameter-details', component: ParameterDetailsComponent

The parameters are handled via the Angular ActivatedRoute service. The calling page, parameters.component.html, links to this page by passing an object of optional parameters to the route like this: ['/parameter-details', {{ '{' }} city: 'Montreal', code:'YUL' {{'}'}}]

The template handles the incoming route values just like properties of an object bound to this component. So the line below is written as: The code for {{city}} is {{code}} where the values between curly braces are substituted for the values passed in via the route.
